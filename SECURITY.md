# 安全策略

## 支持的版本

我们会尽量为**当前 `main` 分支上的最新发布标签或最新提交**提供安全相关修复。请始终使用仓库说明中的方式从源码构建并保持更新。

## 报告漏洞

请 **不要** 在公开 Issue 中披露未修复的安全问题。

推荐方式：

1. 使用 GitHub 的 **Security advisories**（若仓库已开启 *Private vulnerability reporting*，可通过仓库 **Security** 页提交）。
2. 或联系仓库维护者（通过 GitHub 个人主页提供的联系方式，若有）。

请在报告中尽量包含：

- 问题类型与影响范围
- 复现步骤或概念验证（如有）
- 受影响的版本或提交（如已知）

我们会在合理时间内评估并回复；修复进度可能因复杂度而异。
